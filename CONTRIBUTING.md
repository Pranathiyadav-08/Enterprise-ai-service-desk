# Contributing

Thanks for contributing to Enterprise AI Service Desk.

## Getting started

1. Fork the repository and create a branch from the default branch.
2. Set up the backend and frontend using the commands in the [README](README.md).
3. Make one focused change per pull request.
4. Run the relevant checks before opening a pull request.

## Contribution guidelines

- Preserve existing API contracts unless a change is explicitly proposed and documented.
- Do not commit `.env` files, generated FAISS indexes, or private knowledge-base documents.
- Add or update documentation when behavior, configuration, or an endpoint changes.
- Keep code readable, use descriptive names, and follow the conventions already present in the repository.
- Add focused tests when changing backend behavior.

## Pull requests

Include a concise description, the motivation for the change, validation performed, and any relevant screenshots for interface changes. Keep pull requests small and clearly scoped.

## Reporting issues

Use a clear title, steps to reproduce, expected behavior, actual behavior, and environment details. Never include credentials, internal documents, or sensitive user data in an issue.

## Code of conduct

Be respectful, constructive, and inclusive in all project discussions.
